# Boot Sequence Overview

This section explains how Zoho OS gets from GRUB's 32-bit entry environment to the 64-bit kernel entry in `kmain()`.

The visualizer below is the same walkthrough you already created, now embedded into the docs so each page can focus on a specific phase.

## Q&A

**Q: What is the boot sequence?**  
A: It is the ordered path from power-on firmware to the kernel's first C function.

**Q: Why is it split into steps?**  
A: Because each hardware state change depends on the previous one. If the order breaks, the CPU faults or resets.

**Q: What is the end goal?**  
A: Reach `kmain()` in 64-bit mode with paging, memory info, and a valid stack.

## Why this sequence exists

An x86_64 kernel cannot start directly as a normal C program. Before `kmain()` can run, the CPU must be in the right mode, the memory map must exist, and the kernel must have a valid stack.

Think of booting like moving from a temporary workspace into a fully prepared office:

- GRUB gives you the room keys
- the bootstrap code sets up the desk, floor plan, and lighting
- the 64-bit entry code opens the office for normal work

## Diagram

```text
Power on
  -> BIOS / UEFI
  -> GRUB
  -> Multiboot2 handoff
  -> 32-bit bootstrap
  -> PAE + paging setup
  -> 64-bit entry
  -> kmain()
```

## Step map

1. GRUB handoff
2. Multiboot2 header
3. Entry point and stack
4. Enable PAE
5. Load page tables into `CR3`
6. Set `EFER.LME`
7. Enable paging in `CR0`
8. Load the 64-bit GDT and far jump
9. Run 64-bit entry code and call `kmain()`
